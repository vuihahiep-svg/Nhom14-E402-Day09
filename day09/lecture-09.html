Vòng đời kết nối MCP (Discovery & Lifecycle)
Agent tìm thấy năng lực mới như thế nào?
  • 1. Initialization: Client (Agent) kết nối tới Server.
  • 2. Discovery: Server liệt kê các tools, resources và prompts có sẵn.
  • 3. Execution: Agent gọi tool cụ thể với tham số JSON.
  • 4. Sampling: Server có thể yêu cầu Agent thực hiện một hành động (Agent-as-a-service).

State vs message passing — dùng cái nào khi nào?
Shared state cho orchestration · message contract cho giao việc rõ ràng
Trong graph
Shared state
  • giữ context chung của toàn flow
  • hữu ích khi supervisor cần nhìn toàn cục
  • phù hợp với route reason, confidence, status
Ví dụ state: task, risk_high, needs_tool, chosen_worker, retrieved_chunks, final_answer
Qua ranh giới worker
Message contract
  • input/output ổn định để test độc lập
  • dễ version hoá và thay worker
  • rõ lỗi vì có error format nhất quán
Ví dụ message: task, context, constraints, expected_output, error {code, reason}
